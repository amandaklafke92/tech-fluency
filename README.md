# tech-fluency
learning Python and tech fluency from scratch, designed for people like me: non-technical professionals who want to communicate confidently in tech and AI environments without becoming developers. 

## Origin story

It all started with Anthropic's course, AI Fluency, namely, Lesson 2B. In this lesson, one of the exercises suggests starting a conversation with Claude about something you want to learn. Having actively avoided learning to code for so long, for fear it would drag me down yet another, not-immediately-important rabbit hole, I decided this was my chance. A controlled environment, no room for scope creep, just dipping my toes in. 

Turns out, I'm quite the liar. Because here we are, weeks later, with a fully structured tech fluency course in the making.

## What this is

Tech Fluency is a structured course in Python and technology literacy, designed and completed entirely through conversation with Claude. Each session covers new concepts, tests prior knowledge through retrieval practice and grounds technical ideas in real-world professional context.

The course is *not* about becoming a developer: it's about becoming fluent, that is, building enough understanding of how software works to communicate confidently with technical people, ask better questions and operate effectively in tech-driven environments.

## Why

First and foremost, because it's fun. As someone who enjoys solving problems, I don't know how I didn't think of/discover this sooner. And second, because I'm making a deliberate transition into the AI industry. I want to work at an AI company, not in a technical role, but in one that sits close to the technical work: product, operations, strategy, or a role that bridges technical and non-technical teams.

To do that well, I need more than surface-level familiarity with the technology. I need to understand how developers think, what they mean  when they use specific language and how the systems I'll be working  alongside actually function. Tech Fluency is how I'm building that.

It's also an experiment in learning itself: in what it looks like to  use AI not just as a tool for getting answers, but as a genuine teaching  partner, with a structured pedagogy, spaced repetition, and a curriculum  that adapts in real time.

## Who this is for

This project was built for a specific kind of person: me. But as it developed, I realised there are many verions of "me" bobbling about in the world. These "mes" are people who are curious, professionally ambitious and find themselves working closer 
and closer to the technology world, but without a technical background to draw on.

If that sounds like you, Hello, welcome! If I had to guess, you're probably a writer, a strategist, an operator, a marketer or 
a generalist who keeps ending up in rooms full of engineers. You can follow the conversation but not quite participate in it. And you want to change that—not by becoming a programmer, but by becoming someone who genuinely understands what's being built and why.

If that's you, this project might be worth following.

## How it's structured

Each session includes:

- **Opening quiz** — retrieval practice on prior concepts, no notes
- **New concepts** — taught intuition-first, with analogies before syntax
- **Real-world grounding** — each concept anchored in professional 
  context: phrases you'll hear, things you might say, interview 
  applications, and common misconceptions
- **Lesson plan** — a full written record of the session
- **Glossary** — running vocabulary reference, updated each session
- **Field Guide** — a concept-by-concept cheat sheet to show how these concepts sound and look outside of a formal learning context. If you've read Thinking, Fast and Slow, you'll recognise the spirit of the idea: phrases and triggers that prime you to recognise concepts in the real world, not just recall them as abstractions.

## Curriculum

It's still evolving (I'm only up to lesson 4 myself), but so far we have:

**Foundations** ✅
- `print()` and why code runs silently
- Variables and the DRY principle
- Data types: String, Integer, Float, Boolean

**Making things happen** ✅
- Functions: defining, calling, parameters, arguments
- Built-in vs. user-defined functions
- Return values and data flow

**Logic and control** ✅
- Conditionals: `if`, `elif`, `else`
- Boolean checks and comparison operators

**Coming up**
- Loops
- Reading and writing data — files, APIs
- AI-specific literacy — models, training, how prompts connect to code
