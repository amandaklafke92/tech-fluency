# Python Vocabulary — Lesson 1

*Introduction to Python*

> **How to use this:** This glossary grows with every lesson. Use it as a quick-reference when a technical term comes up in conversation and you want to place it fast. Terms are grouped by lesson.

---

| Term | Type | Definition & Example |
|---|---|---|
| **Python** | *noun* | A programming language. Dominant in AI and data science. Reads almost like plain English, which makes it a good starting point for beginners. |
| **Code** | *noun* | Instructions written in a programming language that a computer can follow. Code runs silently — nothing is shown to anyone unless you explicitly tell it to. |
| **Function** | *noun* | A named command that does something. You call it by writing its name followed by parentheses. Think of it as a verb in the language of code. `print("Hello!")` |
| **print()** | *function* | The most fundamental function in Python. Displays something on screen. Without it, code runs invisibly. Used constantly — by beginners and senior engineers alike. `print("Hello, World!")` |
| **Syntax** | *noun* | The rules of a programming language — where brackets go, when to use quotes, capitalisation, etc. Like grammar, but the computer is extremely unforgiving about it. |
| **Output** | *noun* | What appears as a result of running code. In a terminal, it shows in an output box. In an app, it might be text on screen. Code and output are always two separate things. |
| **Variable** | *noun* | A named container that stores a value. Define it once, use it everywhere. If the value needs to change, you update it in one place and it applies throughout the code. `name = "Amanda"` |
| **= (assignment)** | *operator* | In Python, `=` means "store this value in this variable." It is not mathematical equals. Think of it as an arrow pointing left — "put this into that." `age = 28` |
| **String** | *data type* | Text. Always wrapped in quote marks. Python treats anything in quotes as a string — even if it looks like a number. Strings can be displayed but not calculated with. `name = "Amanda"` |
| **Integer** | *data type* | A whole number, no decimal point. No quotes. Python can do maths with it. `age = 28` |
| **Float** | *data type* | A number with a decimal point. Short for "floating point number." Used when precision matters — measurements, percentages, coordinates. `height = 1.65` |
| **Boolean** | *data type* | A value that is either `True` or `False` — nothing else. Capital T and F required. Used for decisions and conditions in code. Named after mathematician George Boole. `is_learning = True` |
| **Data type** | *noun* | The category of a value — text, number, decimal, true/false. Python behaves differently depending on type. Mixing them up accidentally is one of the most common sources of bugs. |
| **Bug** | *noun* | An error in code that causes it to behave unexpectedly. Not always a crash — sometimes it just gives you the wrong result. Finding and fixing bugs is called debugging. |
| **DRY** | *principle* | "Don't Repeat Yourself." A core programming principle. If something appears in multiple places, find a way to define it once. Variables are one of the first tools that let you do this. |
| **+ operator** | *operator* | Does different things depending on data type. With numbers: addition. With strings: joins them together (called concatenation). This is why data types matter. `"Hello, " + name + "!"` |
