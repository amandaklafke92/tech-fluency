# Return Values

*Lesson 2*

**Keywords:** `return value` · `output` · `pipeline` · `data flow` · `input → process → output`

---

**YOU'LL HEAR THIS WHEN** *you're reviewing an AI or data pipeline*

A developer says: *"That function isn't returning the right value"* or *"We're not capturing the output of that step."*

You might be in a data or AI team review as a product manager, analyst, or operations lead. A result was produced but not caught — or the wrong thing came back out.

---

**IN AN INTERVIEW, IF ASKED** *about process design or systems thinking*

*"I think in terms of inputs and outputs — what goes in, what comes out, and whether the output of one step becomes the input of the next. That's how I approach both process design and understanding technical systems."*

This maps directly to the Input → Process → Output model that underpins all software. It works for any role.

---

**SO WHAT?** *the one-line reason this matters to you*

Return values are the plumbing of software — how data flows from one part of a system to another. When people talk about 'pipelines,' this is the mechanism underneath.
