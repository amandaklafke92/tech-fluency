# Data Types

*Lesson 1*

**Keywords:** `string` · `integer` · `float` · `boolean` · `type error` · `type mismatch`

---

**YOU'LL HEAR THIS WHEN** *you're working with data in any role*

A developer or data analyst says: *"The numbers are being treated as text"* or *"There's a type error."*

This happens constantly with imported spreadsheets, form inputs, and database fields — in operations, marketing, and sales just as much as in engineering. You now know what caused it and roughly what needs to happen to fix it.

---

**DEV-TO-ENGLISH** *translation layer*

*"There's a type error"* → Python (or another language) was expecting one kind of data and got another. Not a logic problem — the wrong ingredient went in.

---

**IN AN INTERVIEW, IF ASKED** *about attention to detail or working with data*

*"I've learned that a lot of data issues come down to type mismatches — a number stored as text, a date formatted inconsistently. I now think about data categories, not just data values, when reviewing anything that feeds into a system."*

---

**SO WHAT?** *the one-line reason this matters to you*

A huge proportion of software bugs come down to a type mismatch somewhere. Knowing this exists makes you sharper in conversations about data quality and system errors.
